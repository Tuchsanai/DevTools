ฝึก git + Docker-in-Docker + SSH + FastAPI  |  ทำจริง บันทึกผลจริงทุกขั้นตอน

เอกสารนี้สร้างจากการรันคำสั่งจริงทั้งหมด + แคปหน้าจอจริง (8 ภาพ) — ใช้เป็นแนวทางให้เพื่อนทำตามได้ทันที

📌 กติกาสำคัญ (จาก LAB_GUIDE): ทุก LAB ต้องทำ ผ่าน SSH เท่านั้น — เปิด container แล้ว ssh student@localhost -p 2222 เข้าไปทำงาน (อย่าใช้เทอร์มินัลที่ attach ตรง ๆ หรือ docker exec). ผลลัพธ์ทุกอันในหน้านี้เก็บจากเซสชัน SSH จริง.

0) เตรียมเครื่อง + Build & Run Container ✓ สำเร็จ▲ บนสุด

🎯 โจทย์

บน Windows + Docker Desktop (เปิด WSL2 backend) ให้ build image จาก Dockerfile แล้ว run container แบบ detached พร้อม --privileged (จำเป็นสำหรับ Docker-in-Docker) และ map พอร์ต 2222→22 (SSH) กับ 8000→8000 (Web API).

💻 คำสั่ง (PowerShell บน Windows)
# build image
docker build -t devtools-base-lab:24.04 .

# run แบบ detached (backtick ` = ขึ้นบรรทัดใหม่ใน PowerShell)
docker run -dit --privileged --name devtools-lab `
  -p 2222:22 `
  -p 8000:8000 `
  -v dind-data:/var/lib/docker `
  devtools-base-lab:24.04

📝 หมายเหตุ / จุดพลาดที่พบบ่อย
  • ลืม --privileged → dockerd ใน container จะสตาร์ทไม่ขึ้น (DinD ใช้ไม่ได้)
  • ใช้ -dit (detached) เพราะเราจะเข้าไปทำงานผ่าน SSH ไม่ใช่ attach ตรง
  • ถ้าพอร์ต 2222/8000 ถูกใช้อยู่ จะ error port is already allocated → เปลี่ยนเลขพอร์ตฝั่งซ้าย
  • backtick ` ต้องเป็นตัวท้ายบรรทัดจริง ๆ (ห้ามมีช่องว่างตามหลัง)

LAB 1 — SSH เข้า container + ตรวจเครื่องมือ ✓ สำเร็จ▲ บนสุด

🎯 โจทย์

เชื่อมต่อจาก PowerShell เข้า container ด้วยผู้ใช้ student รหัสผ่าน student (host พอร์ต 2222 → container พอร์ต 22) แล้วตรวจว่ามี git และ Docker-in-Docker ใช้งานได้.

💻 คำสั่ง (PowerShell → เข้าสู่เซสชัน SSH)
ssh student@localhost -p 2222
# ครั้งแรกถาม fingerprint ให้พิมพ์ yes แล้วใส่รหัสผ่าน: student
# หลัง login เปลี่ยนรหัสผ่านด้วย: passwd
💻 คำสั่ง (ในเซสชัน SSH — ตรวจเครื่องมือ)
git --version
docker run --rm hello-world        # ทดสอบ Docker-in-Docker
✅ ผลลัพธ์จริง — login สำเร็จ
LOGIN_OK
student                 # whoami
af2ac04518cd            # hostname (= container id)
git version 2.43.0

📝 หมายเหตุ / จุดพลาดที่พบบ่อย
  • เจอ Host key verification failed เมื่อ rebuild container ใหม่ → ลบ key เก่าด้วย ssh-keygen -R "[localhost]:2222"

  • อย่าทำ LAB ผ่าน docker exec — กติกาบังคับให้ทำผ่าน SSH

LAB 2 — Web API ด้วย FastAPI ✓ สำเร็จ▲ บนสุด

🎯 โจทย์

สร้าง REST API ด้วย Python (FastAPI + uvicorn) ในเซสชัน SSH แล้วเปิดดูจากเบราว์เซอร์บน Windows ที่พอร์ต 8000 รวมถึงหน้า Swagger UI อัตโนมัติ.

💻 ขั้นที่ 1–2: โปรเจกต์ + virtual environment + ติดตั้งไลบรารี
cd ~
mkdir webapi && cd webapi
python -m venv venv
source venv/bin/activate            # prompt จะขึ้น (venv) นำหน้า
pip install fastapi uvicorn

Ubuntu 24.04 ใช้นโยบาย PEP 668 — ห้าม pip install ลงระบบตรง ๆ ต้องใช้ venv

✅ ผลลัพธ์จริง — venv + เวอร์ชันที่ติดตั้ง

Python 3.12.3

💻 ขั้นที่ 3: สร้างไฟล์ main.py
cat > main.py <<'PY'
from fastapi import FastAPI

app = FastAPI(title="DevTools Web API")

@app.get("/")
def home():
    return {"message": "Hello DevTools! Web API ทำงานแล้ว"}

@app.get("/students/{name}")
def greet(name: str):
    return {"student": name, "status": "ok"}
PY
💻 ขั้นที่ 4: รัน API ที่พอร์ต 8000
uvicorn main:app --host 0.0.0.0 --port 8000

ต้องใช้ --host 0.0.0.0 เพื่อให้เข้าถึงจากนอก container ได้ (อย่าใช้ 127.0.0.1)

📸 หลักฐานภาพ (screenshot จริง)
📝 หมายเหตุ / จุดพลาดที่พบบ่อย
  • รัน --host 127.0.0.1 → เปิดจาก Windows ไม่ได้ (connection refused) ต้องเป็น 0.0.0.0
  • ลืม source venv/bin/activate แล้ว pip install → โดน error externally-managed-environment (PEP 668)
  • ภาษาไทยใน JSON เป็น UTF-8 ปกติ — ถ้าเห็นตัวอักษรเพี้ยน มักเป็นเรื่อง encoding ของหน้าจอ/เทอร์มินัล ไม่ใช่ API ผิด
  • อยากให้เซิร์ฟเวอร์รันค้างแล้วทดสอบ curl ต่อ → เปิด SSH อีกหน้าต่าง หรือรันแบบ background

LAB 3 — Docker-in-Docker (build + run ซ้อน) ✓ สำเร็จ▲ บนสุด

🎯 โจทย์

เขียน Dockerfile เล็ก ๆ แล้ว docker build + docker run ภายใน container ให้สำเร็จ (พิสูจน์ว่า Docker daemon ที่ซ้อนอยู่ทำงานครบวงจร).

💻 คำสั่ง (ในเซสชัน SSH) — สร้าง Dockerfile
mkdir ~/dindlab && cd ~/dindlab
cat > Dockerfile <<'DOCKER'
FROM alpine:3.20
RUN echo "build step: greeting baked into image" > /hello.txt
CMD ["sh", "-c", "cat /hello.txt && echo 'Hello from a container built INSIDE the lab container (DinD)!' && uname -a"]
DOCKER
💻 คำสั่ง — build + run
docker build -t my-dind-demo:1.0 .
docker run --rm my-dind-demo:1.0
docker images

📝 หมายเหตุ / จุดพลาดที่พบบ่อย
  • hostname ใน output (7f7bc4072696) เป็นคนละค่ากับ container หลัก — ยืนยันว่ามันคือ container ใหม่ ที่ถูกสร้างจาก daemon ซ้อน

🧹 หยุด / ลบ container▲ บนสุด

💻 คำสั่ง (PowerShell)
docker stop devtools-lab           # หยุด container
docker start devtools-lab          # เปิดใหม่ (แล้ว ssh เข้าไปใหม่)
docker rm -f devtools-lab          # ลบ container
docker volume rm dind-data         # ลบข้อมูล DinD
  • หยุดแล้วเปิดใหม่ด้วย docker start ข้อมูลใน home ของ student ยังอยู่
  • ลบ volume dind-data = ล้าง image/container ที่อยู่ใน DinD ทั้งหมด (เริ่มใหม่หมด)
