ภารกิจ:

2. ทำ LAB จริงตามลำดับ: build/run container → LAB 1 (SSH) → LAB 2 (FastAPI)

3. ใช้ Computer Use เก็บ "หลักฐานภาพ" (screenshot) ของแต่ละขั้นตอนจริง
   บันทึกไฟล์ภาพไว้ในโฟลเดอร์ ./screenshots/ ตั้งชื่อสื่อความ เช่น
   - 01_docker_desktop.png   → เปิด Docker Desktop เห็น container devtools-lab รันอยู่ (สถานะ Running)
   - 02_powershell_run.png   → หน้าต่าง PowerShell ตอน docker build / docker run สำเร็จ
   - 03_ssh_login.png        → เซสชัน SSH ที่ login เป็น student สำเร็จ (whoami, git --version)
   - 04_hello_world.png      → ผล docker run --rm hello-world (Docker-in-Docker)
   - 05_uvicorn_running.png  → log uvicorn ตอนเซิร์ฟเวอร์รันที่พอร์ต 8000
   - 06_browser_root.png     → เบราว์เซอร์เปิด http://localhost:8000 เห็น JSON ตอบกลับ
   - 07_swagger_ui.png       → หน้า Swagger UI ที่ http://localhost:8000/docs
   - 08_dind_build_run.png   → docker build + docker run ของอิมเมจที่สร้างซ้อนใน DinD
   ขั้นตอนการ capture:
   - เปิดโปรแกรม/หน้าต่างจริง (Docker Desktop, PowerShell, Chrome) ด้วย computer use
   - จัดหน้าต่างให้เห็นผลลัพธ์ชัด แล้วถ่าย screenshot เก็บไฟล์จริง
   - ถ้า capture ไม่ได้/ไม่มีสิทธิ์ ให้บันทึกตามจริงว่าใช้วิธีสำรองอะไร (เช่น คำสั่ง screenshot ของ OS)
     และยังต้องมี output ข้อความจริงประกอบเสมอ — ห้ามสร้างภาพปลอม

   *** คุณภาพภาพ (สำคัญมาก — เครื่องนี้เป็นจอ ultrawide / กว้างพิเศษ) ***
   - ห้ามแคปทั้งจอ (full virtual screen) เพราะหน้าต่างจะลอยอยู่กลางพื้นที่ว่างกว้าง ดูไม่สวย/อ่านยาก
   - ต้อง "โฟกัสและ crop เฉพาะหน้าต่างเป้าหมาย" ให้ภาพมีแต่หน้าต่างนั้นเต็มเฟรม:
     1) เปิด/โฟกัสหน้าต่าง (COM AppActivate ด้วย PID — เลี่ยง Win32 P/Invoke ที่ AMSI มักบล็อก)
     2) หา bounding rectangle ของหน้าต่างด้วย UI Automation (managed API ไม่ใช่ P/Invoke):
        [System.Windows.Automation.AutomationElement] -> Current.BoundingRectangle
        (โหลด UIAutomationClient, UIAutomationTypes)
     3) แคปเฉพาะกรอบนั้นด้วย System.Drawing.Graphics.CopyFromScreen โดยส่ง
        ตำแหน่ง/ขนาด = ค่าจาก BoundingRectangle (ได้ภาพ crop พอดีหน้าต่าง)
   - ทำให้หน้าต่างใหญ่และเนื้อหาชัดก่อนแคป:
       * PowerShell: เปิดด้วย -WindowStyle Maximized และตั้งฟอนต์ใหญ่/คอนทราสต์สูง
       * Chrome: เปิดเต็มจอ/ขยายหน้าต่าง และซูมหน้าให้อ่านง่าย
   - หลังแคปทุกภาพ ให้ "เปิดไฟล์ภาพมาตรวจ" ว่า crop พอดีหน้าต่าง อ่านข้อความออก ไม่มีพื้นที่ดำว่าง/
     ไม่มีหน้าต่างอื่นบังหรือแอบโผล่ ถ้ายังไม่สวยให้แคปใหม่จนผ่าน
   - ถ้า BoundingRectangle ใช้ไม่ได้ ให้ fallback: maximize หน้าต่างแล้ว crop เป็นกรอบจอหลัก
     (primary monitor) แทนการใช้ virtual screen ทั้งหมด

4. สร้างไฟล์ lab_result.html แบบ self-contained
   (CSS อยู่ในไฟล์, เปิดด้วยเบราว์เซอร์ได้เลย ไม่พึ่ง internet)

   - ใต้ภาพมีคำบรรยายสั้น ๆ ว่าภาพนี้พิสูจน์อะไร
