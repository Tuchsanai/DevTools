Centralized vs Distributed VCS

อ้างอิงสไลด์หน้า 30–33 · VCS มี 2 รูปแบบหลัก

🏛️ Centralized VCS (รวมศูนย์)

นักพัฒนาทุกคนเชื่อมต่อเข้า Centralized Repository เพียงที่เดียว

  • Single point of failure — ถ้าเซิร์ฟเวอร์กลางพัง ทุกคนหยุดชะงัก
  • Requires constant connectivity — ต้องต่อเน็ตตลอดเวลาจึงทำงานได้
E.G — Subversion (SVN), Endevor

🌐 Distributed VCS (กระจาย)

นักพัฒนาแต่ละคนมี Local Repository เต็มก้อนบนเครื่องตัวเอง และ sync กับ Remote Repository

  • Each developer has a local copy — มีสำเนาเต็ม ทำงาน offline ได้
  • More scalable than centralized — รองรับทีมขนาดใหญ่ได้ดีกว่า
E.G — Mercurial, Git

🍎 macOS / 🐧 Linux

การตั้งค่า Git (Configure)

อ้างอิงสไลด์หน้า 46–50 · Day 1 — Configure Git

ก่อนเริ่มใช้งาน ต้องตั้ง username และ email ซึ่งจะถูกผูกติดกับทุก commit ที่เราสร้างขึ้น

# ตั้งค่าชื่อและอีเมล (--global = ใช้กับทุก repo บนเครื่อง)
git config --global user.name "user"
git config --global user.email "email"

สลับไปใช้ GitHub อีก account

git config --global user.name "user"
git config --global user.email "email"
git config --global credential.username "user"

ลบ config เก่า (เมื่อ commit ไม่ได้ หรือต้องการเปลี่ยน account)

-- ลบ config เก่า --
git config --global --unset user.name
git config --global --unset user.email
git config --global --unset credential.username

-- ลบ origin คนเก่า ถ้า commit ไม่ได้ --
git remote remove origin

ตรวจสอบการตั้งค่า

git config --list              # หรือ  git config -l
git config --list --show-origin  # ดูว่าค่าแต่ละตัวมาจากไฟล์ไหน
💡 ค่า global เก็บที่ไฟล์ ~/.gitconfig · ส่วนค่า local เก็บที่ .git/config ของแต่ละ repo

git status

การ Ignore ไฟล์ด้วย .gitignore

อ้างอิงสไลด์หน้า 62–63 · Ignoring Files

เราสามารถบอก Git ว่าไฟล์/โฟลเดอร์ไหน ไม่ต้อง เก็บเข้า repo โดยสร้างไฟล์ชื่อ .gitignore ไว้ที่ root ของ repository ซึ่งมีประโยชน์กับไฟล์ที่เรา ไม่มีวันอยากจะ commit

ไฟล์ที่ควรใส่ใน .gitignore

🔑

Secrets & Keys

API keys, credentials, รหัสผ่าน

💻

OS Files

ไฟล์ของระบบ เช่น .DS_Store บน Mac

📜

Log Files

ไฟล์ log ต่าง ๆ

📦

Dependencies & Packages

เช่น node_modules ที่ติดตั้งใหม่ได้

ตัวอย่าง Pattern

สร้างไฟล์ชื่อ .gitignore ที่ root แล้วเขียน pattern ข้างใน:

# .gitignore
.DS_Store        # ignore ไฟล์ชื่อ .DS_Store
folderName/      # ignore ทั้งไดเรกทอรี (ลงท้ายด้วย /)
*.log            # ignore ทุกไฟล์ที่นามสกุล .log
💡 สร้าง .gitignore สำเร็จรูปตามภาษา/framework ได้ที่ → toptal.com/developers/gitignore

Private Repository & Personal Access Token

อ้างอิงสไลด์หน้า 64–70 · Day 1 — Private Repositories and Tokens

repo สาธารณะ clone ได้ตามปกติ แต่ repo ส่วนตัว (private) ต้องยืนยันตัวตนด้วย Personal Access Token (PAT) แทนรหัสผ่าน

# แบบเดิม (repo สาธารณะ)
git clone https://github.com/account/repo.git

# แบบใช้ PAT (repo ส่วนตัว)
git clone https://username:YOUR_TOKEN@github.com/username/repo.git

ขั้นตอนสร้าง Personal Access Token

1
คลิก รูปโปรไฟล์ มุมขวาบน → เลือก "Settings"
2
ใน sidebar ซ้าย เลื่อนลงไปเลือก "Developer settings"
3
เลือก Personal access tokens → Tokens (classic)
4
คลิก "Generate new token" → ตั้งชื่อ, เลือกอายุ (expiration), และ scope สิทธิ์ที่ต้องการ (เช่น repo)
5
คัดลอก token เก็บไว้ทันที — token จะแสดงเพียงครั้งเดียว!
⚠️ อย่า commit token ลง repo เด็ดขาด ควรใส่ไว้ใน .gitignore และถ้า token รั่วไหลให้ revoke แล้วสร้างใหม่ทันที
